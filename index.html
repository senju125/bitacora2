Position

Ideas principales

  • position: static; Los elementos HTML se posicionan estáticos de forma predeterminada. Los elementos posicionados estáticos no se ven afectados por las propiedades top, bottom, left, right y z-index.
  • position: relative; Un elemento con position: relative; está posicionado en relación con su posición normal.
  • El posicionamiento relativo desplaza un elemento respecto de su posición original establecida mediante el posicionamiento normal.
  • El desplazamiento de la caja se controla con las propiedades top, bottom, left, right y z-index. Los demás elementos no ocuparán el espacio que deja el elemento desplazado.
  • position; absolute; El posicionamiento absoluto se emplea para establecer de forma exacta la posición en la que se muestra en la caja de un elemento.
  • La nueva posición del elemento se indica mediante las propiedades top, bottom, left, right y z-index. que en este caso depende del posicionamiento del elemento contenedor cuando tiene una posición diferente a static.
  • Las propiedades: top, bottom, left y right se utilizan para posicionar el elemento. Un elemento fijo no deja un espacio en la página donde normalmente se ubicará.
  • postion: sticky; Se posiciona en función de la posición de desplazamiento del usuario.

Apuntes

  • Cada posición tiene una función especifica, algunas para se ajustan dependiendo del proyecto.
  • Cuando algún elemento está establecido con posición static, puede ser que no se vea afectado por otros "position"

Resumen

En resumen podemos establecer que las propiedades position deben tomarse en cuenta, ya que por defecto algunos elementos ya tienen aplicada la propiedad static, y esta puede afectar al uso de otras position.

Display

Ideas principales

  • Propiedad display: block; Siempre comienza en una nueva línea y ocupa todo el ancho disponible (se extiende hacia la izquierda y hacia la derecha tanto como sea posible).
  • Propiedad display: inline; Un elemento inline no comienza en una nueva línea y solo ocupa el ancho que sea necesario. Ej: span a img
  • Propiedad display: inline-block; Permite establecer un ancho y una altura en el elemento. Además, se respetan los margin y padding, que con display: inline no son respetados. No agrega un salto de línea después del elemento, por lo que el elemento puede alinearse junto a otros elementos.
  • Propiedad display: none; Se usa comúnmente en JavaScript para ocultar y mostrar elementos cuando el usuario realiza alguna acción como pulsar un botón o un enlaces sin eliminarlos ni crearlos nuevamente. Hace que el elemento no genere ninguna caja. El resultado es que el elemento desaparece por completo de la pagina y no ocupa sitio, por lo que los elementos adyacentes ocupan su lugar.
  • Propiedad display: table; Permite asignar un comportamiento de tabla a los elementos que hay dentro (anidados a la etiqueta donde estas colocando el display: table). Dentro de este elemento de pueden usar otra series de valores de display que corresponden con los roles de distintas etiquetas HTML que usas en las tablas.

Apuntes

  • El mejor para utilizar es el display inline-block, permite controlar ancho y alto, además respeta paddings y margin.
  • Se puede usar el display none, para hacer elementos desplegables que no ocupan espacio hasta que el usuario lleva acabo alguna acción.
  • El display table permite insertar tablas a los elementos anidados para organizar información.

Resumen

En resumen, las propiedades display permiten organizar y enlistar los elementos de una página web de diferentes maneras, así como insertar tablas y ocultar elementos desplegables mediante acciones o botones.

Nuevas propiedades de CSS

Ideas principales

Las propiedades nuevas de Css son:

  • Transform, que modifica la forma de un elemento con scale, rotate, skew y translate.
  • Border image, que crea un borde personalizado alrededor de las imagenes.
  • Outline, que crea un borde alejado del borde original de un elemento.
  • HSLA, son los valores tono, saturacion y luminosidad, aunado al valor A, que es el canal Alfa que le agrega opacidad.
  • Radial gradient, que aplica un gradiente Radial, se le aplica color, circle y ellipse.
  • Linear gradient, aplica un gradiente lineal, se le aplica un color solido al fondo, los atributos indican el punto inicial y los colores usados.
  • @font-face, esta regla permite usar cualquier fuente que necesitemos, declarandola en css y despues asignandole nombre a la fuente.
  • Box-shadow, esta propiedad le da sombra a las cajas, tiene 5 parametros, desplazamiento vertical y horizontal, color, difuminación e inset.
  • Border-radius, redondea las esquinas de las cajas.
  • Text-shadow, le adjunta una sombra personalizable a los elementos, tiene 4 parametros, color, desplazamiento horizontal y vertical y el valor de difuminación.

Apuntes

  • Hay que tener en cuenta el orden de los atributos a modificar en los gradient.
  • Se añaden nuevas funciones que tal vez no todos los navegadores soporten, tal vez tiene que tener un plan b en caso de que no sea soportado.
  • Se tiene que tener en cuenta todos los factores para usar correctamente las nuevas propiedades.

Resumen

Las funciones de Css se expanden mucho y dejan más a la imaginación, se extienden en sombras y estilos nuevos.

Animación

Ideas principales

  • Css permite animar la mayoria de elementos en html sin usar JavaScript, con la regla @keyframes.
  • su sintaxis es:
  • @keyframes nombre { from{background-color:red; width: 100px; } to{background-color: yellow; width:500px; }
  • Para que una Animación funcione, se debe vincular a un elemento, por ejemplo:
  • div{ animation-name:anim01; }
  • @keyframes nombre { from{background-color:red; width: 100px;
  • Sus propiedades son: linear, ease, ease-in, ease-out, ease-in-out y cubic-bezier.
  • Tambien: animation-iteration-count, animation-delay, animation-play-state.

Apuntes

  • Para poder animar correctamente primero hay que declarar y despues vincularla a un elemento.
  • Hay manera de hacer que las animaciones tengan un ciclo definido, o infinito.
  • Las animaciones hacen distintas acciones segun la que se le adjunte.

Resumen

Las animaciones son una rama extensa de Css y hay que tener cierto grado previo de conocimiento para poder aplicarlas correctamente, ya que hay algunos pasos que no se pueden intuir al ser una funcion nueva.

Referencias avanzadas

Ideas principales

  • Este tema trabaja las siguientes pseudo clases: nth-child(), first-child, last-child, only-child y not(),
  • nth-child, permite encontrar a un "hijo" especifico.
  • first-child, referencia solo al primer hijo de todos los que haya.
  • last-child, referencia solo al ultimo hijo de todos.
  • only-child, referencia a un elemento siempre y cuando sea el unico hijo disponible.
  • not(), nos permite declarar una excepción en los hijos, así saltandolos al aplicarselo.
  • Tambien cuenta con selector de relacion (+), de hermanos adyacentes, selecciona un hermano inmediato, es decir que está justo después de otro.
  • Selector de hijos directos (>), que selecciona los elementos que son hijos directos de un elemento padre.
  • Selector de atributo - [atributo*="cadena"], selecciona elementos cuyo atributo contiene una cadena específica.

Apuntes

  • Con este tipo de pseudo clases el trabajo del programador es mucho más facil y rapido.
  • Siempre hay que tener en cuenta que pseudo clase estamos usando y no confundirla, porqué tienen nombres parecidos.
  • Este tipo de relaciones y selectores permiten manipular de manera más eficaz nuestra página para personalizarla.

Resumen

Podemos establecer que las relaciones, pseudoclases y selectores, son herramientas muy utiles que reducen el trabajo y el tiempo empleado por el programador.
